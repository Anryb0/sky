# Sky host
**Общее описание**
**\
`	`Система представляет собой прототип облачного хостинг-провайдера. Пользователи смогут арендовать себе VDS серверы через веб-интерфейс и управлять ими в личном кабинете. Виртуальные машины будут запушены на локальном сервере. 

**Используемое программное и аппаратное обеспечение**

На локальном сервере установлена операционная система Ubuntu Server 24.04. Работа с виртуальными машинами осуществляется встроенным в эту ОС гипервизором KVM (Kernel-Based Virtual Machine). Управление гипервизором будет осуществляться набором инструментов libvirt. Для установки на виртуальные машины пользователям будет предложен выбор из следующих ОС: Ubuntu Server и Debian.

Управление серверами будет осуществляться пользователями через веб-сайт. Для разработки front-end сайта будет использоваться библиотека React.js. Back-end будет разработан на языке PHP. Данные о пользователях и услугах хранятся в базе данных MySQL. Имитация платежей на сайте будет осуществляться с использованием платежной системы ЮКасса. Сайт будет работать на арендованном VDS со статичным IP адресом. 

Кроме того, для управления серверами из интернета необходимо иметь белые (статичные) IP адреса для каждой созданной виртуальной машины. Их получение не представляется возможным. У локального сервера тоже нет своего статичного IP адреса, поскольку провайдер не предоставляет такую услугу. Данную проблему планируется решать следующим образом: на арендованном VDS сервере будет создана VPN сеть (OpenVPN или другая), к которой автоматически подключаются виртуальные машины после установки на них операционной системы. Пользователь также подключается к этой сети и получает доступ к виртуальной машине через выданный ему внутренний адрес ВМ. 

**Пользовательский сценарий**

1) Пользователь заходит на главную страницу сайта, где он может ознакомится с предоставляемыми услугами
1) Пользователь переходит на страницу, где описаны различные предлагаемые ему тарифные планы (количество CPU ядер, объем оперативной памяти и дискового пространства: различные ресурсы, выделяемые под виртуальную машину на локальном сервере). Он регистрируется (с подтверждением по e-mail) и выбирает подходящий для своих задач план. 
1) Пользователь выбирает нужную операционную систему.
1) Пользователь перенаправляется на сайт платежной системы, осуществляет оплату за выбранные услуги. 
1) После оплаты пользователь получает доступ к файлу с VPN конфигурацией, отображается страница с инструкцией по подключению к VPN. Создается виртуальная машина и на нее устанавливается операционная система без вмешательства пользователя с помощью конфигурационного файла Preseed. 
1) Пользователю в личном кабинете выдается IP адрес виртуальной машины, пароль от root пользователя. Также доступна вкладка для управления ВМ: ее можно включить, перезапустить или остановить. Есть возможность продлить оплату/отказаться от услуги.
